## Wall-clock results (median ms)

Generated by **`scripts/benchmark_cli.sh`**:

| Scenario | Step | Median (ms) |
|----------|------|-------------:|
| **A** | `sxmc stdio "sxmc serve --paths tests/fixtures" skill_with_scripts__hello` | **11** |
| **B** | `sxmc api <petstore openapi> --list` | **715** |
| **B** | `sxmc api … findPetsByStatus status=available` | **1024** |
| **B** | `curl` only to same Petstore endpoint | **448** |
| **C** | `sxmc stdio "sxmc serve --paths tests/fixtures" --list` | **11** |
| **D** | `sxmc scan --paths tests/fixtures --skill malicious-skill` | **12** |
| **Micro** | Local OpenAPI file + ephemeral `127.0.0.1` server + `sxmc api … listPets` | **15** |

**Interpretation:** Petstore steps are **network-dominated**. **A / C / D / Micro** show that local MCP bridge + scan paths stay in the **~10–15 ms** range (median on this machine).
